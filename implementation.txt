Vitalis AI - Virtual Healthcare Assistant Implementation Plan
Build a Minimum Viable Product (MVP) for Vitalis AI, a secure, virtual healthcare assistant. The system will ingest medical documents, perform OCR and data extraction, store information in a vector database, and use a RAG pipeline with strict safety guardrails to provide dietary and medical insights based on static knowledge and patient history. The frontend will be a cohesive, 3-panel desktop React/Next.js application.

User Review Required
IMPORTANT

Based on your request for the best free tools to make the project functional first, I have selected the following stack for the MVP:

OCR Engine: PyTesseract (Free, open-source, runs locally. Perfect for an MVP before scaling to paid cloud services).
Vector Database: ChromaDB (Free, open-source, runs entirely locally, requires no cloud setup).
LLM Provider: Groq API (with Llama 3) or Google Gemini API. Both offer excellent, highly capable free tiers. Alternatively, if you want 100% data privacy for medical records, we can use Ollama to run models completely locally (requires local installation).
